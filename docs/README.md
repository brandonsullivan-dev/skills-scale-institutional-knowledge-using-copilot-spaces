# OctoAcme Project Management Docs

This README is the central entry point for OctoAcme project management guidance. It summarizes the core lifecycle and links to each detailed process document.

## Project Management Lifecycle at a Glance

OctoAcme uses an iterative lifecycle:

1. **Initiation**: Validate the problem, define goals and success metrics, and align stakeholders.
2. **Planning**: Build a prioritized, estimable backlog with milestones, risks, dependencies, and clear Definition of Done.
3. **Execution & Tracking**: Deliver in small increments, follow team rhythm and PR/QA workflows, and track delivery/impact metrics.
4. **Release & Deployment**: Ensure readiness checks, deploy safely, verify outcomes, and communicate releases.
5. **Retrospective & Improvement**: Capture learnings, assign actions, and continuously improve delivery.

## Key Principles

From the [Project Management Overview](./octoacme-project-management-overview.md), OctoAcme emphasizes:

- Customer-first outcomes
- Iterative delivery in small, testable increments
- Clear ownership across PM, Product, Engineering, QA, and stakeholders
- Data-informed decisions
- Psychological safety and continuous learning
